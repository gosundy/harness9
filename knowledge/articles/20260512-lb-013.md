# Running Subagents in the Background

> **来源**: [langchain_blog](https://www.langchain.com/blog/running-subagents-in-the-background) | **热度**: 10 | **重要性**: 值得了解（6/10）

## 摘要

Deep Agents 引入的后台异步子 Agent 能力解决了多 Agent 系统中的关键工程问题：内联子 Agent 执行会阻塞主流程并中断中间更新，限制了多 Agent 协作的实际并发效率。通过将子 Agent 推入后台异步执行并保留主 Agent 的动态控制权，Deep Agents 实现了真正意义上的 Agent 级并发。这一能力与 harness9 引擎层的并发工具执行设计（使用 sync.WaitGroup + 独立超时）在理念上高度一致，侧面验证了并发执行作为高效 Agent 系统核心能力的行业共识。对于正在设计多 Agent 编排框架的团队，此文提供了一个成熟的异步子 Agent 设计参考。

## 核心亮点

- 异步后台子 Agent 解决了内联 Agent 阻塞主流程的核心痛点，多 Agent 并发效率大幅提升
- 主 Agent 实时控制子 Agent 生命周期（启动/引导/取消），赋予编排层动态调度能力
- 此设计与 harness9 并发工具执行架构理念高度一致，是工程上可相互印证的架构方向
